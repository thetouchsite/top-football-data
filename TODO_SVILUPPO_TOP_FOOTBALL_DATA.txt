LINEE GUIDA (COSA E' IL PROGETTO ORA)
====================================================================

- Feed calcio: un solo provider dati, Sportmonks Football API v3.
- Sportradar: rimosso dal repository (nessun pacchetto, nessun fallback, nessun client).
- OddsMatrix: mai integrato come prodotto; lo stub e i riferimenti in health/catalog sono stati
  rimossi; quote e comparatore restano da costruire sopra Sportmonks (e UI interna).
- ID canonico match: fixture id Sportmonks (`/match/[id]`, API fixtures, watchlist/search).
- Health (`GET /api/health`): readiness su MongoDB, Stripe e `providers.sportmonks` soltanto.
- `GET /api/football/odds/futures`: ancora placeholder fino a integrazione outrights Sportmonks.
- Dati Live (pagina `/dati-live`, nav, fetch livescore su altre schermate): controllati da
  `NEXT_PUBLIC_FEATURE_DATI_LIVE=true` (vedi `src/lib/feature-flags.js` e README). Default spento.
- Immagini Sportmonks (loghi / foto): implementate end-to-end su provider+domain+UI (`FootballMediaImage`,
  campi `home_media` / `away_media` / `league_media` e `player.media`). Base CDN configurabile con
  `SPORTMONKS_MEDIA_BASE_URL`. Le schermate non devono leggere oggetti raw provider per le immagini.
  Contrasto dark mode: le immagini remote usano di default un “mat” chiaro (fondo bianco / zinc-100 in dark)
  + padding interno, così loghi scuri o asset per sfondo chiaro restano leggibili; prop opzionale
  `surface="none"` su `FootballMediaImage` se serve eccezione.
- Area pubblica marketing (aprile 2026): completata con landing unica `/come-funziona` come hub principale
  (sezioni Funzioni, Vantaggi, Premium, Telegram, FAQ) e menu ad anchor interni sticky. Rimosse le pagine
  separate `/premium-offerta`, `/faq`, `/telegram`; copy commerciale semplificato lato cliente; card hero
  match resa compatta/minimal con etichetta esplicita `Quote 1X2`. Footer policy legali mantenuto attivo.
- Legal Center / Trust Center (aprile 2026): introdotto hub `/legal` con documenti modulari versionati
  (`src/content/legal/*`), sottoroute canoniche (`/legal/privacy`, `/legal/cookies`, `/legal/terms`,
  `/legal/premium`, `/legal/telegram-bot`, `/legal/processors`, `/legal/disclaimer`, `/legal/contact`,
  `/legal/changelog`), componenti riusabili (`src/components/legal/*`), API richieste privacy
  `/api/legal/requests`, entry point "Gestisci cookie" e changelog legale dinamico. Route legacy
  (`/privacy`, `/cookie`, `/termini`, `/disclaimer`, `/abbonamenti-rimborsi`, `/telegram-policy`)
  mantenute via redirect verso `/legal/*`.

Stato Sezione A (SAFE, implementato ora):
- Payload match con quota modello (1/p), best odds bookmaker e value % quando il feed odds e disponibile.
- Comparatore quote in formato 4 slot con CTA neutra configurabile, senza dipendere da endpoint nuovi.
- UI modelli/match detail con fallback espliciti (`sportmonks_feed_math` vs `fallback_derivato`).
- Confidenza con priorita API e fallback interno composito (`confidence_source`: `sportmonks_api` o `composite_internal`).
- Highlight value bet coerente tra card e dettaglio match su tutti i mercati supportati (1/X/2, O/U, GG/NG).
- Blocco dettaglio "Probabilita e quote" + "xG/Risultati probabili" riallineato visivamente per coerenza UX.
- Modelli (colonna destra) allineata a uso pre-match: Top Value, Confidenza alta, Prossimi match,
  CTA Telegram via `NEXT_PUBLIC_TELEGRAM_URL`, box Premium (rimosso blocco Live center non essenziale).
- Gate per passaggio a FULL: add-on confermati + endpoint reali + affiliazioni/copy legale approvate.

Sintesi macro (da negoziare sul contratto Sportmonks e sul backlog):

A — Modelli / singolo match — Prediction API + quote live; probabilita e quota modello; value;
   comparatore a 4 slot + CTA affiliazione (copy legale); xG e tre risultati esatti piu probabili;
   barre di confidenza; formazioni verso Reliability score; toggle «solo Value Bet»; grafico
   Pressure Index.

B — Analisi statistica deep — Formazioni aggiornate (~60 min), impact player (penalita se assenti
   titolari chiave); player props (tiri, xG, heatmap se API, disciplina); momentum, xPts, pressure,
   tempi di rete; tab SPA (Formazioni / Player stats / Team Momentum); comparatore «intelligente»
   sul tab (es. marcatore vs ammoniti); lazy loading quote al click sul giocatore.

C — Bet multipla — Orchestratore edge/EV su 3-4 eventi con soglie; UI confidence / Data edge /
   probabilita di successo; sostituire placeholder futures; notifiche (Telegram/push) su EV > soglia;
   pagina performance storiche (ROI) e sintesi canale Telegram.

D — Trasversali — Copy CTA conforme (alternativa a «Gioca ora» se richiesto); GEO/SEO; allineamento
   Premium/MultiBet alle feature vendute; area pubblica/marketing se in scope.

ARCHITETTURA RILEVANTE (PUNTATORI)
====================================================================

- Catalogo competizioni: `src/lib/competitions/catalog.js`
- Domain: `src/lib/domain/*`
- Provider Sportmonks: `src/lib/providers/sportmonks/index.js`
- Orchestrazione feed: `src/server/football/service.js`
- Priorita leghe / ranking: `src/lib/sportmonks-priority-league-ids.js`, ordinamento in service
- Env opzionali feed: `SPORTMONKS_SCHEDULE_LEAGUE_FILTER_STRICT`, `SPORTMONKS_SCHEDULE_LEAGUE_IDS`, ecc.
- Shell/UI: `src/app/layout.jsx`, `src/components/layout/*`, schermate in `src/screens/*`
- Probabilita O/U e GG (schedule/match): calcolo in `src/lib/providers/sportmonks/index.js`
  — priorita (1) predizioni API yes/no se presenti, (2) percentuali modello derivato `buildGoalMarkets`
  (`ouProbDerived` / `ggProbDerived`, stesse formule delle quote derivate), (3) implicite da quote
  bookmaker (`100/quota`), (4) fallback implicite sulle quote mostrate. Campi payload: `ouProb`, `ggProb`.
- Value bet (UI): `src/components/shared/ValueBetBadge.jsx`; dato mostrato da `buildDerivedValueBet` in
  `sportmonks/index.js` (euristica interna) — non e l'API ufficiale Sportmonks "Value bets".
- Value bet Sportmonks ufficiale: add-on Predictions; endpoint `predictions/value-bets` e
  `predictions/value-bets/fixtures/{id}` (fair_odd, odd, bookmaker, is_value, stake). Vedi
  https://docs.sportmonks.com/v3/tutorials-and-guides/tutorials/odds-and-predictions/predictions/value-bet
  — integrazione futura quando il piano cliente lo consente.
- Toaster: `src/components/ui/toast.jsx` usa `pointer-events-none` su viewport/provider per non
  bloccare click su navbar/header (`z` toast sopra la barra).
- Live odds UI (griglia da props, non brand esterno): `src/components/live/LiveOddsGrid.jsx`
- Auth/account: `src/app/api/auth`, `src/lib/auth-*.js`, API `/api/account/*`
- Documentazione operativa sintetica: `README.md`
- Feature flag: `src/lib/feature-flags.js`
