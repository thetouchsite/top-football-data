TOP FOOTBALL DATA
MAPPA CHIARA API vs CALCOLI (BASATA SU docs/cliente/funzioni-piattaforma.md)

Scopo:
- Distinguere cosa deve arrivare da API.
- Distinguere cosa va calcolato internamente.
- Evidenziare cosa puo funzionare in modalita mista (API + calcolo).

Legenda:
- API = richiesto principalmente da feed esterno (Sportmonks).
- CALCOLO = logica matematica/prodotto nostra.
- API + CALCOLO = dato base da API, poi trasformazione/score interno.

====================================================================
SEZIONE A — MODELLI PREDITTIVI SINGOLO MATCH
====================================================================

| Funzione | Tipo | Nota semplice |
|----------|------|---------------|
| Probabilita 1X2 / U-O / GG-NG | API | Il cliente chiede prediction API come fonte primaria. |
| Quota modello (1 / probabilita) | CALCOLO | Formula interna diretta dal valore di probabilita. |
| Value bet % (book quota vs quota modello) | API + CALCOLO | Quote bookmaker da API, confronto matematico nostro. |
| xG squadra | API + CALCOLO | Base da API; se incompleta puo usare fallback derivato dichiarato. |
| Top 3 risultati esatti | API + CALCOLO | Priorita API predictions; fallback modello interno se mancanti. |
| Comparatore quote 4 slot | API + CALCOLO | Quote/API bookmaker + regola prodotto (top value + 3 reference). |
| CTA comparatore | CALCOLO | Regola UI/business interna con link affiliato configurato. |
| Highlight visuale value (box quota) | CALCOLO | Mappa interna valueBet->mercato/esito per evidenziare il riquadro corretto in lista e dettaglio. |
| Barre di confidenza | API + CALCOLO | Se API da confidence/reliability presente si usa; altrimenti score composito interno. |
| Reliability score da formazioni | API + CALCOLO | Formazioni da API; scoring affidabilita calcolato internamente. |
| Toggle “solo Value Bet” | CALCOLO | Filtro frontend interno su campi value disponibili. |
| Pressure index (radar/istogramma) | API + CALCOLO | Si basa su xG/forme API + formula/visualizzazione interna. |

====================================================================
SEZIONE B — ANALISI STATISTICA DEEP DATA
====================================================================

| Funzione | Tipo | Nota semplice |
|----------|------|---------------|
| Formazioni aggiornate (~60 min) | API | Dato live ufficiale richiesto dal cliente. |
| Impact player (assenza titolari) | API + CALCOLO | Assenze da API, penalita probabilistica da modello interno. |
| Player props (tiri, xG, disciplina) | API | Fonte principale da endpoint/statistiche giocatore. |
| Heatmap giocatore | API | Solo se endpoint/lega lo supporta. |
| Team momentum | API + CALCOLO | Eventi/statistiche da API + indice sintetico interno. |
| xPts (expected points) | API + CALCOLO | Da metriche API e normalizzazione formula interna. |
| Tempi di segnatura | API + CALCOLO | Eventi goal da API + aggregazione e pattern interno. |
| Tab SPA analisi | CALCOLO | Esperienza frontend interna, non dipende da API dedicate. |
| Comparatore context-aware (marcatore/ammoniti) | API + CALCOLO | Mercato corretto da API + logica di contesto interna. |
| Lazy loading quote giocatore | CALCOLO | Ottimizzazione frontend/backend interna. |

====================================================================
SEZIONE C — MULTIBET ORCHESTRATORE
====================================================================

| Funzione | Tipo | Nota semplice |
|----------|------|---------------|
| Probabilita evento base | API + CALCOLO | Priorita prediction API, fallback modello interno se necessario. |
| Edge singolo | CALCOLO | Formula interna: probabilita x quota. |
| EV totale multipla | CALCOLO | Prodotto edge dei singoli eventi. |
| Selezione 3-4 eventi con soglia | CALCOLO | Regole interne prodotto. |
| Confidence score multipla | CALCOLO | Score sintetico interno su EV/rischio/copertura. |
| Data edge % | CALCOLO | Trasformazione matematica dell’edge. |
| Probabilita successo multipla | CALCOLO | Combinazione probabilita eventi selezionati. |
| Comparatore multipla 4 slot | API + CALCOLO | Quote bookmaker API + ranking/CTA interno. |
| Notifiche EV > soglia | API + CALCOLO | Trigger interno su score, invio via canale esterno. |
| Performance storiche / ROI | CALCOLO | Storico interno (DB) + formule ROI/report. |

====================================================================
BLOCCO TRASVERSALE — IMMAGINI (CDN / MEDIA)
====================================================================

| Funzione | Tipo | Nota semplice |
|----------|------|---------------|
| URL logo squadra / competizione / foto giocatore | API | Path/URL derivati dai nodi inclusi nel feed Sportmonks (participant, league, player, ecc.). |
| Normalizzazione `media.imageUrl` / `thumbUrl` | API + CALCOLO | Mapping stabile in `sportmonks/index.js` + default vuoti in domain; base CDN sovrascrivibile (`SPORTMONKS_MEDIA_BASE_URL`). |
| Fallback UI (iniziali / placeholder) | CALCOLO | `FootballMediaImage.jsx` — nessuna immagine finta remota; solo fallback locale se URL assente o errore load. |
| Contrasto loghi su tema scuro | CALCOLO | Stesso componente: contenitore “light mat” dietro URL remoti (default), bordo leggero; opzione `surface="none"`. |

====================================================================
BLOCCO TRASVERSALE — COME LAVORARE SENZA “FAKE DATA”
====================================================================

1) Quando API esiste:
- usare API come primaria;
- mostrare chiaramente la sorgente.

2) Quando API manca:
- usare fallback interno dichiarato;
- etichettare in UI/documentazione che e fallback.

3) Quando la feature e mista:
- separare sempre "dato ricevuto" da "calcolo prodotto";
- mantenere formule tracciabili e semplici da verificare.

Riferimento ufficiale requisiti:
- docs/cliente/funzioni-piattaforma.md
